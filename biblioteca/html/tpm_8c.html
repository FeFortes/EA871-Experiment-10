Detailed Description

Date
Aug 3, 2017
Author
Wu Shin-Ting e Leandro Cavalcanti Silva
Note
Veja a relação dos campos dos registradores na Figura 31-1 de KL-25 Sub-Family Reference Manual Configuração dos módulos TPM0, TPM1 e TPM2 Periodo_{UP} = (MOD+1) * (PS/(frequência de source clock)) (CPWMS=0); Periodo_{UP-DOWN} = (MOD) * (PS/(frequência de source clock)) * (1/(1+CPWMS)) (CPWMS=1). Fórmula geral: Periodo_{TPMS} ~ (MOD) * (PS/(frequência de source clock)) * (1/(1+CPWMS)) frequência de MCGFLLCLK = 20.971520MHz Vale observar que para programar um duty cycle 100%, MOD < 0xFFFF (Seção 31.4.6)

initialize a TPM module

Parameters
[in]modulomudule ID
[in]prepré-scale

< TPMSRC=SIM_SOPT2[25:24]=01 (MCGFLLCLK|MCGPLLCLK/2)

< PLLFLLSEL=SIM_SOPT2[16]=0 (MCGFLLCLK)

< TPM0=SIM_SCGC6[24]=1 (Clock gate de TPM0 habilitado)

< TPM0=SIM_SCGC6[25]=1 (Clock gate de TPM1 habilitado)

< TPM0=SIM_SCGC6[26]=1 (Clock gate de TPM2 habilitado)

< TOF=TPMx_TOF[7]: w1c (limpa a pendncia do TPM0)

< CMOD=TPMx_SC[4:3]=0b01 (incrementa a cada pulso do LPTPM)

< PS=TPMx_SC[2:0]=0b100 (Fator Pré-escala=16)

< CPWMS=TPMx_SC[5]=1 (modo de contagem crescente-decrescente)

< CPWMS=TPMx_SC[5]=0 (modo de contagem crescente)

< DMA=TPMx_SC[8]=0 (desabilita DMA)

< TOIE=TPMx_SC[6]=0 (desabilita interrupção "overflow")

< COUNT=TPMx_CNT[15:0]: w (qq acesso de escrita reseta LPTPM)

< MOD=TPMx_MOD[15:0]=COUNTER_OVF atualizado conforme CMOD e CPWMS

< (Seção 31.4.8.1 de KL25 Sub-Family Reference Manual)

< CROT=TPMx_CONF[18]=0 (não recarrega o valor de LPTPM no gatilho)

< CSOO=TPMx_CONF[17]=0 (continua contagem em overflow)

< CSOT=TPMx_CONF[16]=0 (inicia contagem quando habilitado)

< GTBEEN=TPMx_CONF[9]=0 (LPTPM contador como base de tempo)

< DOZEEN=TPMx_CONF[5]=0 (continua contagem em hibernação)

< DBGMODE=TPMx_CONF[7:6]=0b11 (continua contagem em Debug)
